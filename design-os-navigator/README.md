# Design OS Navigator

**A visual navigator for Design OS projects** — track your project from discovery to code with a spec-driven methodology.

## Features

### Navigator
Visualize your Design OS project structure as an interactive graph:
- **Global view** of modules, specs, and files
- **Readiness score** per module and overall
- **Quick navigation** to specs, screens, and code
- **One-click commands** (`/spec`, `/build`, `/review`...)

### Prototyper
Manage artifacts generated by Claude:
- **Artifact feed** with preview
- **Quick copy** of generated code
- **Regeneration** with variants or custom prompts

### Console
Launch Claude Code directly from VS Code/Cursor with your project context.

## Installation

### Prerequisites

- **VS Code 1.85+** or **Cursor**
- **[Claude Code](https://docs.anthropic.com/en/docs/claude-code)** installed (terminal version)

### Option 1: Clone the full Design OS repo (recommended)

This method gives you everything: the extension + all skills + project templates.

```bash
git clone https://github.com/MehdiVilquin/Design-Os.git
cd Design-Os
```

Then install the extension:
1. Open the project in **Cursor** (or VS Code)
2. Press `Shift + CMD + P` (Mac) or `Shift + Ctrl + P` (Windows/Linux)
3. Type **"Extensions: Install from VSIX"**
4. Navigate to `design-os-navigator/design-os-navigator-0.7.1.vsix`
5. Click **Install**

### Option 2: Install VSIX only (if you already have the skills)

If you already have a Design OS project with skills set up:

1. Download `design-os-navigator-0.7.1.vsix` from releases
2. In VS Code/Cursor: `Shift + CMD + P` → **"Extensions: Install from VSIX"**
3. Select the `.vsix` file

## Getting Started

1. **Install Claude Code** (if not already done)
   ```bash
   npm install -g @anthropic-ai/claude-code
   ```

2. **Open your project** in Cursor/VS Code

3. **Click the Design OS icon** in the activity bar (left sidebar)

4. **Click "Launch console"** to open Claude Code terminal

5. **Type `/onboarding`** to set up your project

## Project Structure

The extension works with the Design OS template structure:

```
your-project/
├── CLAUDE.md                    ← Project instructions
├── .claude/
│   ├── context.md               ← Active module & intent
│   ├── profile.md               ← User profile
│   └── skills/                  ← Claude Code skills (required)
│       ├── onboarding/          ← /onboarding
│       ├── discovery/           ← /discovery
│       ├── ux-design/           ← /ux
│       ├── spec/                ← /spec
│       ├── build/               ← /build
│       ├── review/              ← /review
│       └── ...
├── 01_Product/                  ← Product artifacts
├── 02_Build/                    ← Source code
├── 03_Review/                   ← Conformity reviews
└── 04_Lab/                      ← Prototypes
```

**Important:** The skills are co-dependent with the extension — they form an integrated system.

## Design OS Methodology

Design OS is a spec-driven framework in 3 phases:

```
Discovery → Spec → Build → Review
```

| Phase | Purpose |
|-------|---------|
| **Discovery** | Understand users and domain |
| **Spec** | Validated specifications before any code |
| **Build** | TDD code from specs |
| **Review** | Code vs spec conformity |

## Available Commands

| Command | Description |
|---------|-------------|
| `/onboarding` | Set up your project |
| `/discovery` | Product discovery workshop |
| `/ux` | UX co-creation and hypothesis validation |
| `/spec` | Generate spec from EPICs |
| `/build` | TDD code from validated spec |
| `/review` | Code vs spec conformity score |
| `/health` | Project health diagnostic |
