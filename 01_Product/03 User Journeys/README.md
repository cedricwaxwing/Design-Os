# 03 User Journeys

> User journey visualizations per module. Generated by `/ux` (step 3.7).

---

## Available Formats

### User Journey (SVG)
Column-based overview: each EPIC = one column, user stories listed below.
Viewable directly in Cursor/VS Code.

```
journey-[name].svg
```

**Useful for**: EPIC overview, story → screen mapping, team presentations.

### User Flow (Mermaid)
Flowchart diagram of user journey with decisions and branches.

```
flow-[name].md
```

**Useful for**: detailing a specific journey, documenting edge cases, showing decision branches.

## Structure

```
03 User Journeys/
└── {module}/
    ├── journey-onboarding.svg       ← EPIC column view
    ├── journey-collaboration.svg
    ├── flow-creation-study.md       ← Mermaid flowchart
    └── flow-review-process.md
```

## How to Generate

1. Run `/ux` on your module
2. After Screen Map creation (step 3.5), the agent offers to generate visualizations
3. Choose the appropriate format:
   - **SVG** for an overview
   - **Mermaid** for a detailed journey
   - **Both** if you want both perspectives

## Templates

Two templates are available to get started or understand the expected format:

- **`_template-journey.svg`** — SVG template for column-based user journey (EPICs → stories). Open in Cursor/VS Code to see the visual render.
- **`_template-flow.md`** — Mermaid template for user flow with decisions, branches, and edge cases.

These templates show the expected structure. Actual journeys/flows are generated automatically by `/ux` (step 3.7) or can be created manually.

## Alternative Formats

In addition to SVG and Mermaid, you can also:
- Export a diagram from Figma/FigJam and place it here as `.png` or `.svg`
- Use other tools (Whimsical, Miro) and export the result
- Describe the flow in plain text in a `.md` file — `/spec` and `/ux` agents will read it too

SVG is the recommended format as it's versionable (Git), viewable in the IDE, and can be generated by agents.

## This folder is empty?

Normal if you haven't run `/ux` yet. User journeys are generated during UX exploration, after Screen Map creation.
