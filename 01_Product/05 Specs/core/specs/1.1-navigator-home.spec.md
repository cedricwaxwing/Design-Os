## 1. User story

As a **Template user** (designer/PM), I want to **see the project flow graph and global readiness at a glance** so that I can **decide which phase to run next and open a node for details**.

**Route**: N/A (webview main view)  
**Screen Map reference**: `00_screen-map.md` — Navigator (home)  
**Stories covered**: User sees project readiness at a glance; user runs a phase command (entry point).

---

## 2. Acceptance criteria (Gherkin)

- [ ] **Given** the Design OS Navigator is open **When** the view loads **Then** the flow graph shows all phase nodes (Material, Strategy, Discovery, UX, Spec, Build, Review, Lab, etc.) with their maturity label and gate ratio.
- [ ] **Given** the Navigator tab is active **When** the user clicks a node **Then** the node detail panel opens on the right with status, actions, and files for that node.
- [ ] **Given** the Navigator is visible **When** the user clicks "Launch console" **Then** the configured CLI terminal is shown (or a prompt to choose claude/other).
- [ ] **Given** the graph data failed to load **When** the user is on the Navigator **Then** an error state is shown with a retry or message (e.g. "No workspace" or "Webview not built").
- [ ] **Given** the user is on the Prototyper tab **When** they click the Navigator tab **Then** the Navigator home view is shown (flow graph + footer).

## 3. Layout and composition

```
+------------------------------------------------------------------+
|  Design Os    [Prototyper] [Navigator]        [Launch console]  |
+------------------------------------------------------------------+
|                                                                  |
|   Flow graph (nodes + edges)                                     |
|   Material → Strategy → Discovery → [Design System, UX, Lab]     |
|   → Spec, UI → Build → Review                                    |
|   Each node: label, maturity tag, gate dots, ratio               |
|                                                                  |
+------------------------------------------------------------------+
|  AI — Material → … → Review   [====    ] 80%  12 nodes · 2 files |
+------------------------------------------------------------------+
```

**Reference**: `wireframes/board-navigator-e2e.svg` (frame 1).

**Zones**: Topbar (title, tabs, Launch console), main (flow graph), footer (readiness bar + stats).

---

## 4. Data

**Source**: Extension provides `initialData` (and `graphData` via postMessage) with `nodes`, `edges`, `context`, `globalReadiness`.

- **Nodes**: `{ id, label, phase, readiness, maturity, gates, fileCount, files, commands, dependsOn, unlocks, … }`
- **Edges**: `{ from, to, type? }`
- **globalReadiness**: number 0–100

No REST API; data is injected by the host. Error state when `graphData` is null or workspace unavailable.

---

## 5. Design system

- **Layout**: Flex column; topbar fixed height (e.g. 48px), main flex-grow, footer fixed height. Graph area uses available space.
- **Tokens**: `bg-base`, `bg-surface`, `border-default`, `text-primary`, `text-secondary`; primary for active tab and primary CTA; spacing `space-4` / `space-6` between zones.
- **Responsive**: Desktop-only for MVP; webview is embedded in the IDE.
