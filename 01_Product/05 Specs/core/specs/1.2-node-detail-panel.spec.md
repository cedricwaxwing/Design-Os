## 1. User story

As a **Template user** (designer/PM), I want to **see a node’s status, gates, actions, and files in a side panel** so that I can **run a phase command, open a file, or preview it without losing sight of the graph**.

**Route**: N/A (panel, no route)  
**Screen Map reference**: `00_screen-map.md` — Node detail panel  
**Stories covered**: User runs a phase command; user opens a file from the panel; user closes the panel to return to the graph.

---

## 2. Acceptance criteria (Gherkin)

- [ ] **Given** a node is selected **When** the panel is open **Then** the panel shows the node label, maturity, file count, gates (met/total), and at least one primary action (e.g. run command).
- [ ] **Given** the panel is open **When** the user clicks a "Run command" action **Then** the command is sent to the host (Cursor/terminal per config) and the panel remains open unless the design specifies closing.
- [ ] **Given** the panel lists files **When** the user clicks a file **Then** the file is opened in the editor (or preview for supported types).
- [ ] **Given** the panel is open **When** the user clicks the close control (×) or re-clicks the same node **Then** the panel closes and the graph view is fully visible again.
- [ ] **Given** no node is selected **When** the user is on the Navigator **Then** the panel is closed (or hidden). Selecting a node opens the panel with that node’s data.

## 4. Data

**Source**: Selected node from `graphData.nodes` — same object as in Navigator home.

- **Node**: `{ id, label, maturity, readiness, fileCount, files[], gates[], commands[], dependsOn[], unlocks[], … }`
- **files**: `{ path, name, status? }[]`
- **commands**: `{ command, label, description }[]`

No separate API; panel reflects the currently selected node. If selection is cleared, panel closes or shows empty state per product rule.

---

## 5. Design system

- **Layout**: Panel fixed width (300px), full height; sections stacked vertically with clear headings (STATUS, ACTIONS, FILES, CONNECTIONS); spacing `space-4` between sections.
- **Tokens**: `bg-surface`, `bg-elevated` for panel; `border-default`; `text-primary`, `text-secondary`; primary token for primary CTA; `radius-md` for buttons.
- **Responsive**: Desktop-only for MVP.
