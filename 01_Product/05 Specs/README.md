# 05 Specs — Validated Specifications

> Specs generated by `/spec`. Source of truth for `/build`.

---

## Structure

```
05 Specs/
├── _templates/                    ← Spec templates (cross-module)
│   ├── component-spec.md         ← For a component (modal, form, card...)
│   └── page-spec.md              ← For a complete page (dashboard, list, detail...)
│
└── {module}/                      ← Specs per active module
    ├── 00_screen-map.md          ← N stories → M screens mapping (source of truth)
    ├── specs/                     ← Spec files
    │   ├── 1.1-overview-page.spec.md
    │   ├── 1.2-detail-modal.spec.md
    │   └── 2.1-filter-sidebar.spec.md
    └── screens/                   ← SVG visual references
        ├── overview.svg
        └── detail.svg
```

## Two available templates

| Template | When to use |
|----------|-------------|
| **component-spec.md** | Reusable component: modal, form, card, sidebar |
| **page-spec.md** | Complete page with route: dashboard, list, detail page |

Both have 9 mandatory sections (in VALIDATED mode).

## Spec modes

| Mode | Sections | TBD allowed | Accepted by /build |
|------|----------|-------------|-------------------|
| **VALIDATED** | 9 mandatory | No | Yes |
| **DRAFT** | 3 minimum (Story + Criteria + Layout) | Yes | No |

DRAFT mode allows rapid iteration during exploration phase. To move to VALIDATED, complete all 9 sections and eliminate all TBDs.

## Naming convention

```
{X.Y}-{name}.spec.md
```

- **X** = EPIC number
- **Y** = story/screen number within EPIC
- **name** = short description in kebab-case

Examples: `1.1-overview-page.spec.md`, `2.3-user-modal.spec.md`

## Screen Map

The `00_screen-map.md` file is the central pivot. It maps N user stories to M actual screens.

**Rule**: `/spec` ALWAYS consults the Screen Map before creating a spec. If a story isn't mapped, `/spec` blocks and redirects to `/ux`.

## This folder is empty?

Normal. Run `/ux` to create the Screen Map, then `/spec` to generate specs.
