# 04 Ideation — Idea Vault

> Creative brainstorming with mandatory persistence. All ideas are written — retained, discarded, parked — with their reasoning.

---

## Principle

**Zero judgment during free phase, structure after.** Everything said is written, nothing disappears with the session.

## Structure

```
04 Ideation/
└── {module}/
    └── ideation-log.md       ← Append-only, generated by /ideate
```

Each module has its own `ideation-log.md`. Ideas are added in append-only mode (never deleted).

## Tag system

Each idea receives a tag that traces its lifecycle:

| Tag | Meaning | Reasoning required |
|-----|---------|-------------------|
| `IDEA` | Raw, not yet evaluated | No |
| `EXPLORED` | Discussed, with context and consequences | No |
| `RETAINED` | Selected for /ux or /spec | **Yes** — why this idea |
| `DISCARDED` | Consciously rejected | **Yes** — why not |
| `PARKED` | Good idea, not now | **Yes** — condition for revival |

**Transitions**: `IDEA` -> `EXPLORED` -> `RETAINED` | `DISCARDED` | `PARKED`

## How to generate

### Option 1 — With the agent (recommended)
- **`/ideate`** — Complete brainstorm: free divergence then structured convergence
- **`/ideate quick`** — Quick idea capture without sorting phase
- **`/ideate review`** — Review of parking lot (`PARKED` ideas)

### Option 2 — Manually
Create an `ideation-log.md` file in the module subfolder and follow the format:

```markdown
## Session — {YYYY-MM-DD} — {subject}

### Ideas

- **[IDEA]** {description}
- **[RETAINED]** {description}
  > Reasoning: {why}
- **[DISCARDED]** {description}
  > Reasoning: {why not}
```

## Impact on agents

Agents consult this folder for:
- **`/ux`** — Feed solution exploration with raw ideas and discarded paths
- **`/spec`** — Trace reasoning behind design choices ("Decisions" section)
- **`/review`** — Verify that `RETAINED` ideas are properly implemented
- **`/o`** — Insert `/ideate` between Discovery and UX in the flow

## This folder is empty?

Normal if you haven't run `/ideate` yet. Brainstorming typically happens after Discovery and before UX exploration.
