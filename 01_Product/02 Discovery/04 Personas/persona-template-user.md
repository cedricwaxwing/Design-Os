## Role

Designer or product lead who will clone the Design OS core template to start new design projects. They run discovery, define screens, write or commission specs, and need a repeatable flow.

## Key Need

A single, lightweight template that gives structure (personas, brief, screen-map, specs) without heavy process, so they can go from idea to validated spec (and optionally to build) quickly.

## Primary Pain

Starting each project from zero: redefining personas, re-explaining the domain, and losing the link between screens, specs, and code. No shared “source of truth” for what’s in scope.

## Context of Use

- **Frequency**: Per project (weekly or monthly starts).
- **Device**: Desktop (Cursor/VS Code, browser for Figma/design).
- **Environment**: Solo or small team; async collaboration.

## Goals

1. Bootstrap a new project in under an hour (clone template, set module, run /discovery or /ux).
2. Keep screens, specs, and build aligned via one screen-map and one readiness view.
3. Choose speed (MVP) or depth (Epic) without changing the framework.

## Frustrations

1. Recreating the same structure (folders, brief, personas) every time.
2. Specs and designs drift; no clear “this screen = this spec”.
3. Unclear what’s “done” (no gates/readiness) until deep in build.

## Quote

> “I want to open a new project, run a few commands, and have a clear path from problem to spec to code.”

## Technical Proficiency

Medium to high — comfortable with markdown, optional code (React/TS if they use /build). Willing to use Cursor/VS Code and follow a spec-driven flow.
