# 03 Research Insights

> Patterns, hypotheses, and conclusions from user research.

---

## What to put here

| Type | Description |
|------|-------------|
| **Interview syntheses** | Common patterns across multiple interviews |
| **Competitive analysis** | Competitor strengths/weaknesses |
| **Validated/invalidated hypotheses** | Research conclusions |
| **Opportunity Map** | Product opportunity mapping |
| **Jobs-to-be-Done** | What users are trying to accomplish |

## Recommended format

```markdown
# Insight — [Insight Title]

**Source**: [interviews / analytics / benchmark / survey]
**Confidence**: HIGH / MEDIUM / LOW
**Date**: [YYYY-MM-DD]

## Finding
[Factual description of what was observed]

## Product Impact
[How this influences product design]

## Evidence
- [Interview X — verbatim]
- [Data point Y]

## Recommendation
[Action to take in design]
```

## Naming convention

```
[YYYY-MM-DD]-insight-[theme].md
```

Example: `2026-02-20-insight-notification-overload.md`

## Impact on agents

- **`/ux`** — Insights feed design hypotheses (step 2)
- **`/spec`** — Identified constraints end up in acceptance criteria
