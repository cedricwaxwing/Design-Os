# 03 Research Insights — Syntheses de recherche

> Patterns, hypotheses et conclusions tirees de la recherche utilisateur.

---

## Quoi mettre ici

| Type | Description |
|------|------------|
| **Syntheses d'interviews** | Patterns communs entre plusieurs entretiens |
| **Analyse concurrentielle** | Forces/faiblesses des concurrents |
| **Hypotheses validees/invalidees** | Conclusions de la recherche |
| **Opportunity Map** | Cartographie des opportunites produit |
| **Jobs-to-be-Done** | Ce que les utilisateurs essaient d'accomplir |

## Format recommande

```markdown
# Insight — [Titre de l'insight]

**Source** : [interviews / analytics / benchmark / survey]
**Confiance** : FORT / MOYEN / FAIBLE
**Date** : [YYYY-MM-DD]

## Constat
[Description factuelle de ce qu'on a observe]

## Impact produit
[Comment ca influence le design du produit]

## Evidence
- [Interview X — verbatim]
- [Data point Y]

## Recommandation
[Action a prendre dans le design]
```

## Convention de nommage

```
[YYYY-MM-DD]-insight-[theme].md
```

Exemple : `2026-02-20-insight-notification-overload.md`

## Impact sur les agents

- **`/ux`** — Les insights nourrissent les hypotheses de design (etape 2)
- **`/spec`** — Les contraintes identifiees se retrouvent dans les acceptance criteria
